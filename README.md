## 1. ERC-4337 over EOA

ERC-4337은 이더리움 프로토콜 자체를 수정하지 않고, 스마트 컨트랙트 기반의 계정 추상화를 구현하는 표준입니다. 계정 추상화란 EOA의 고정된 인증 방식과 실행 로직을 스마트 컨트랙트 코드로 대체함으로써, 지갑의 동작 방식을 프로그래머블하게 만드는 개념입니다. ERC-4337은 기존 이더리움 트랜잭션 흐름 바깥에 별도의 인프라 레이어를 구축하며 UserOperation, Bundler, EntryPoint, Paymaster, SCA 등의 구성요소를 가집니다. UserOperation은 ERC-4337의 기본 실행 단위입니다. 기존 EOA 트랜잭션과 달리, 사용자의 의도를 담은 객체로서 표준 멤풀이 아닌 별도의 대체 멤풀에서 처리됩니다. Bundler는 여러 UserOperation을 수집하여 하나의 온체인 트랜잭션으로 묶어 제출하는 노드입니다. 번들러는 가스비를 선납하고, 이후 EntryPoint 컨트랙트를 통해 정산받습니다. 번들러의 존재로 인해 사용자는 직접 ETH를 보유하지 않아도 트랜잭션을 발행할 수 있습니다. EntryPoint 컨트랙트는 ERC-4337 생태계의 핵심입니다. 모든 UserOperation의 검증과 실행을 조율하며, 이더리움 재단이 감사한 단일 표준 컨트랙트가 네트워크 전체에서 공유됩니다. Paymaster는 사용자 대신 가스비를 납부하는 스폰서 컨트랙트입니다. 이를 통해 DApp이 자신의 사용자를 위한 가스를 대납하거나, 사용자가 USDC 등 ERC-20 토큰으로 가스를 지불하는 것이 가능해집니다. SCA는 사용자의 실제 지갑 컨트랙트입니다. 인증 로직, 실행 로직, 복구 로직이 모두 코드로 정의되어 있어 개발자가 원하는 방식으로 커스터마이징할 수 있습니다.

ERC-4337은 기존의 EOA와 다르게 소셜 복구가 가능합니다. 신뢰할 수 있는 지인이나 기기를 가디언으로 등록해두면, 개인키를 분실하더라도 가디언의 승인을 통해 계정 접근권을 복구할 수 있습니다. 이는 Web2의 계정 복구 경험과 유사한 수준의 안전망을 제공합니다. 추가적으로 ERC-4337은 다양한 서명 방식을 지원합니다. ECDSA 외에도 WebAuthn, 패스키, 멀티시그, BLS 서명 등 다양한 인증 방식을 컨트랙트 코드로 구현할 수 있으며 생체 인증과 블록체인이 직접 연결합니다. 또한 세션 키를 통한 제한적 권한 위임이 가능하여, 게임이나 DeFi 플랫폼에서 매 트랜잭션마다 서명을 요구하는 대신, 특정 시간 또는 금액 범위 내에서만 유효한 세션 키를 발급하여 자동 실행을 허용할 수 있습니다. 이는 Web3 UX를 Web2 수준으로 끌어올리는 핵심 개선점 입니다. 마지막으로 ERC-4337에선 배치 트랜잭션을 단일 UserOperation으로 처리할 수 있습니다. 예를 들어 토큰 승인과 스왑을 두 번의 트랜잭션이 아닌 한 번의 작업으로 실행함으로써 가스비와 사용자 마찰을 최소화 할 수 있습니다.
